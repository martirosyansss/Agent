## 2. FEATURE ENGINEERING — ПОЛНАЯ МАТЕМАТИКА

### 2.1 FeatureVector (28 полей)

#### EMA (Exponential Moving Average)
```
k = 2 / (period + 1)
EMA[0] = SMA(closes[:period])
EMA[i] = price[i] × k + EMA[i-1] × (1 - k)
```
| Параметр | Таймфрейм | Назначение |
|----------|-----------|------------|
| EMA 9 | 1h | Быстрый тренд |
| EMA 21 | 1h | Средний тренд |
| EMA 50 | 4h | Долгосрочный фильтр |

#### RSI (Relative Strength Index — Wilder's Smoothing)
```
deltas[i] = close[i] - close[i-1]
gains[i] = max(delta, 0)
losses[i] = abs(min(delta, 0))

avg_gain[0] = sum(gains[:14]) / 14
avg_loss[0] = sum(losses[:14]) / 14
avg_gain[i] = (avg_gain[i-1] × 13 + gains[i]) / 14
avg_loss[i] = (avg_loss[i-1] × 13 + losses[i]) / 14

RS = avg_gain / avg_loss
RSI = 100 - 100 / (1 + RS)    → диапазон [0, 100]
```
- Период: 14
- Таймфрейм: 1h

#### Stochastic RSI
```
rsi_series = RSI за каждый бар (период 14)
window = последние 14 значений RSI
min_rsi = min(window)
max_rsi = max(window)
stoch_rsi = 100 × (rsi[-1] - min_rsi) / (max_rsi - min_rsi)
```
- Порог overbought: 80
- Порог oversold: 20

#### MACD (Moving Average Convergence Divergence)
```
fast_ema = EMA(close, 12)
slow_ema = EMA(close, 26)
macd_line = fast_ema - slow_ema
signal_line = EMA(macd_line, 9)
histogram = macd_line - signal_line
```
- histogram > 0 → бычий импульс
- histogram < 0 → медвежий импульс

#### ADX (Average Directional Index)
```
h_diff = high[i] - high[i-1]
l_diff = low[i-1] - low[i]
plus_dm = h_diff if (h_diff > l_diff AND h_diff > 0) else 0
minus_dm = l_diff if (l_diff > h_diff AND l_diff > 0) else 0
true_range = max(H-L, |H-prev_close|, |L-prev_close|)

(Wilder smoothing 14 периодов)
plus_di = 100 × smoothed_plus / smoothed_tr
minus_di = 100 × smoothed_minus / smoothed_tr
DX = 100 × |plus_di - minus_di| / (plus_di + minus_di)
ADX = Wilder_EMA(DX, 14)    → диапазон [0, 100]
```
| Значение | Интерпретация |
|----------|---------------|
| < 20 | Слабый тренд (sideways) |
| 20-25 | Переход |
| > 25 | Сильный тренд |
| > 40 | Экстремально сильный |

Таймфрейм: **4h**

#### Bollinger Bands
```
middle = SMA(close[-20:], 20)
stdev = sqrt(variance(close[-20:]))
upper = middle + 2 × stdev
lower = middle - 2 × stdev
bandwidth = (upper - lower) / middle
```
- Таймфрейм: 4h
- Squeeze: bandwidth < 0.05

#### ATR (Average True Range — Wilder's)
```
TR[i] = max(H[i]-L[i], |H[i]-C[i-1]|, |L[i]-C[i-1]|)
ATR[0] = mean(TR[:14])
ATR[i] = (ATR[i-1] × 13 + TR[i]) / 14
```
- Период: 14, Таймфрейм: 4h

#### Объём
```
volume_sma_20 = SMA(volume[-20:], 20)
volume_ratio = current_volume / volume_sma_20
OBV: cumulative — прибавляем volume если close > prev_close, вычитаем если <
```

#### Производные (1h)
```
price_change_Nm = (close - close[-N]) / close[-N] × 100   (N = 1, 5, 15)
momentum = (close - close[-10]) / close[-10] × 100
```

#### News Sentiment (обогащение после build)
```
news_sentiment: float [-1.0 .. +1.0]    — взвешенный overall score
fear_greed_index: int [0 .. 100]         — Fear & Greed Index
news_impact_pct: float                    — средний impact новостей
high_impact_news: int                     — кол-во новостей с |impact| >= 1.5%
```

### 4.6 Grid Trading
**Режим**: sideways (25%)

#### Построение сетки:
```

step = (BB_upper - BB_lower) / 8
levels = [BB_lower + step × i for i in range(9)]
Перестроение: каждые 24 часа
```

#### BUY:
```
Для каждого незаполненного уровня (снизу вверх):
  если price ≤ level → BUY
  SL = price × (1 - 5%)
  TP = price × (1 + 0.3%)
  confidence = 0.70-0.80 (±0.05 от sentiment)
```

#### SELL:
```
PnL ≥ +0.3%  → SELL (profit capture)
PnL ≤ -5.0%  → SELL (stop loss)
```

#### Safety: Если цена выходит за пределы сетки ±2%, стратегия замирает.

## 8. POSITION SIZING

### 8.1 Критерий Келли (Half-Kelly)
```
f = (p × b - q) / b
где:
  p = win_rate
  q = 1 - p
  b = avg_win_pct / avg_loss_pct

f_half = f × 0.5
f_clamped = clamp(f_half, 0.0, 0.25)
```

### 8.2 Volatility Factor (ATR)
```
atr_pct = (ATR / price) × 100
base_atr_pct = 1.5%

volatility_factor = base_atr_pct / atr_pct
volatility_factor = clamp(factor, 0.3, 1.5)
```
Логика: если рынок 2× более волатилен, позиция ×0.5

### 8.3 Regime Dampener (ADX)
```
if ADX ≥ 30:    dampener = 1.0 (полный размер)
elif ADX ≥ 20:  dampener = 0.6 + (ADX-20)/10 × 0.4 (линейно)
else:            dampener = 0.5 (половина)
```

### 8.4 Итоговый расчёт
```
if kelly_fraction > 0.01:
    base_pct = kelly_fraction × 100
    method = "kelly_atr"
else:
    base_pct = 5.0  (fixed minimum)
    method = "fixed"

adjusted_pct = base_pct × volatility_factor × regime_dampener
budget_pct = clamp(adjusted_pct, 2.0%, max_position_pct)
budget_usd = balance × budget_pct / 100
budget_usd = min(budget_usd, max_order_usd)
quantity = budget_usd / price
```

## 9. DYNAMIC SL/TP

### 9.1 Параметры по стратегиям
| Стратегия | ATR SL mult | R:R ratio | Min SL | Max SL |
|-----------|-------------|-----------|--------|--------|
| EMA Cross | 2.0 | 2.0 | 1.0% | 8.0% |
| BB Break  | 1.8 | 2.5 | 1.0% | 8.0% |
| Mean Rev  | 2.5 | 1.5 | 1.5% | 8.0% |
| MACD Div  | 2.2 | 2.0 | 1.0% | 8.0% |
| DCA Bot   | 3.0 | 2.0 | 2.0% | 10.0% |
| Grid      | 1.5 | 1.0 | 0.5% | 8.0% |

### 9.2 Расчёт
```
atr_pct = (ATR / entry_price) × 100

sl_pct = atr_pct × atr_sl_mult
sl_pct = clamp(sl_pct, min_sl, max_sl)

tp_pct = sl_pct × rr_ratio
tp_pct = clamp(tp_pct, min_tp, max_tp)

sl_price = entry × (1 - sl_pct/100)
tp_price = entry × (1 + tp_pct/100)
```

**Пример** (EMA Cross: entry=$67000, ATR=$1400):
```
atr_pct = 2.09%
sl_pct = 2.09 × 2.0 = 4.18% → clamp → 4.18%
tp_pct = 4.18 × 2.0 = 8.36% → clamp → 8.36%
sl_price = $64,199
tp_price = $72,602
```

# 🔰 ОБЪЯСНЕНИЕ ДЛЯ НАЧИНАЮЩИХ (ПРОСТЫМ ЯЗЫКОМ)

> Если вы никогда не программировали и не торговали на бирже — этот раздел для вас.
> Здесь всё то же самое, что описано выше, но понятным языком, без формул.

---

## Что это за бот?

**Sentinel** — это программа-робот, которая сама следит за ценой криптовалюты (Bitcoin и другие монеты на бирже Binance) и принимает решения: **купить**, **продать** или **ничего не делать**.

Бот работает 24/7 без перерывов. Он не спит, не паникует и не жадничает — он действует строго по правилам, которые в него заложены.

---

## Как бот получает информацию?

Представьте, что бот — это трейдер, который одновременно смотрит на несколько экранов:

1. **Графики цены** — бот подключён к бирже Binance через "прямой провод" (WebSocket). Он видит каждую свечу (свеча — это цена за определённый период: 1 минута, 5 минут, 1 час и т.д.). Каждая свеча содержит 4 главных числа: цена открытия, максимум, минимум, цена закрытия. Плюс объём торгов.

2. **Крипто-новости** — бот читает новости с трёх сайтов (CoinDesk, CoinTelegraph, Decrypt). Он не понимает текст сам — он отправляет заголовки в нейросеть (Llama 3.3), которая отвечает: "это хорошая новость для крипты" или "плохая".

3. **Индекс страха и жадности** — это публичный показатель, который измеряет настроение всего рынка по шкале от 0 до 100. Ноль — все в панике и продают, 100 — все в эйфории и скупают.

---

## Что такое "индикаторы"?

Бот не смотрит просто на цену. Он высчитывает **индикаторы** — это математические формулы, которые помогают понять, что происходит на рынке:

- **EMA (скользящая средняя)** — это "сглаженная" линия цены. Если быстрая EMA (за 9 часов) пересекает медленную EMA (за 21 час) снизу вверх — это сигнал, что цена может расти. Как если бы температура на улице начала расти быстрее, чем обычно — значит, скоро потеплеет.

- **RSI (индекс относительной силы)** — показывает, "перекуплен" или "перепродан" актив. Шкала от 0 до 100. Если RSI ниже 30 — монету слишком много продавали, возможно, скоро вырастет (распродажа). Если выше 70 — слишком много покупали, возможно, скоро упадёт.

- **MACD** — показывает скорость изменения тренда. Работает как спидометр: не просто "мы едем", а "мы ускоряемся" или "мы замедляемся".

- **Bollinger Bands (полосы Боллинджера)** — это "коридор", в котором обычно движется цена. Если цена выходит за коридор — это необычное событие, и бот обращает на это внимание.

- **ATR (средний истинный диапазон)** — показывает, насколько "нервный" рынок. Если ATR высокий — цена прыгает туда-сюда, рынок волатильный. Если низкий — рынок спокойный.

- **ADX (индекс направленного движения)** — показывает силу тренда. Если ADX больше 25 — есть чёткий тренд (вверх или вниз). Если меньше 20 — рынок "болтается" без направления.

- **Объём (Volume)** — сколько денег проходит через рынок. Если цена растёт на большом объёме — это сильный сигнал. Если на маленьком — рост может быть ложным.

Всего бот рассчитывает **28 различных показателей** для каждого момента времени.

---

## Как бот принимает решение?

У бота есть **6 стратегий** — это как 6 разных экспертов, каждый со своим подходом. Каждый час (когда закрывается часовая свеча) все 6 экспертов смотрят на одни и те же данные и высказывают своё мнение:

### 1. EMA Crossover + RSI (Трендовый следопыт)
**Логика**: "Если быстрая средняя пересекает медленную вверх И RSI не в зоне перекупленности — покупаем."
- Это как ехать по трассе: если обе полосы показывают "дорога вперёд свободна" — жмём на газ.

### 2. Bollinger Breakout (Ловец прорывов)
**Логика**: "Если цена вылетела за верхнюю полосу Боллинджера на сильном объёме — это прорыв, покупаем."
- Как пробка от шампанского — если давление сильное, пробка вылетает далеко.

### 3. Mean Reversion (Возврат к среднему)
**Логика**: "Если цена упала слишком далеко от среднего — она, скорее всего, вернётся обратно. Покупаем на «распродаже»."
- Как резинка: если её растянуть слишком сильно — она стремится вернуться. Эта стратегия покупает, когда все паникуют.

### 4. MACD Divergence (Охотник за дивергенциями)
**Логика**: "Если цена падает, но MACD начинает расти — значит, скоро разворот вверх."
- Как мяч, который ещё летит вниз, но уже замедляется — скоро отскочит.

### 5. DCA Bot (Автоматическое усреднение)
**Логика**: "Покупаем регулярно фиксированными порциями, не пытаясь угадать идеальный момент. Если цена упала — покупаем больше (дешевле же!), если выросла — покупаем меньше."
- Как копилка: кладёшь деньги каждый месяц, не думая. Со временем средняя цена покупки сглаживается.

### 6. Grid Trading (Сетка)
**Логика**: "Расставляем «сетку» ордеров: покупаем на каждом уровне падения, продаём на каждом уровне роста."
- Как рыболовная сеть: не знаем, где рыба, но сети расставлены везде.

---

## Что такое "уверенность" (confidence)?

Каждая стратегия выдаёт не просто "купить/продать", а ещё число от 0.0 до 1.0 — **уверенность**:
- **0.85** — "я почти уверен, это отличный момент"
- **0.65** — "вроде похоже на сигнал, но не идеально"
- **0.40** — "слабый сигнал" (бот его проигнорирует, порог обычно 0.6)

Новости влияют на уверенность: если стратегия хочет купить и новости позитивные — уверенность растёт на +0.10. Если новости негативные — падает на -0.10. Это как дополнительный голос "за" или "против".

---

## Что такое ML-предиктор?

**ML (Machine Learning)** — это "машинное обучение". Бот смотрит на историю: "в прошлом, когда показатели были похожими, цена росла или падала?" На основе тысяч предыдущих ситуаций он учится предсказывать будущее.

- Используется алгоритм **Random Forest** (случайный лес) — это 100 "деревьев решений", каждое из которых голосует. Итоговый ответ — как голосование: если 70 из 100 деревьев сказали "вырастет" — предиктор даёт 70% вероятности роста.
- Предиктор работает как **фильтр**: он не генерирует сигналы сам, а проверяет сигналы от стратегий. Если стратегия сказала "покупай", а предиктор говорит "вероятность роста всего 40%" — сигнал блокируется.
- Порог: предиктор пропускает сигнал, только если вероятность правильного направления ≥ 55%.

---

## Как бот защищается от потерь?

Это самая важная часть. Бот имеет **многоуровневую защиту**, как сейф с несколькими замками:

### Уровень 1: Размер позиции (сколько покупаем)
Бот никогда не ставит всё на одну сделку. Он рассчитывает размер по формуле:
- Берёт процент от баланса (обычно 2-10%)
- Умножает на "коэффициент Келли" (математическая формула для оптимального размера ставки)
- Уменьшает, если рынок слишком волатильный (ATR высокий)
- Уменьшает, если тренд слабый (ADX низкий)

**Результат**: даже если сделка убыточная, потеря — это маленький процент от общего баланса.

### Уровень 2: Stop-Loss и Take-Profit (автоматический выход)
Для каждой сделки бот заранее ставит два уровня:
- **Stop-Loss** — "если цена упала на X%, автоматически продаём и фиксируем убыток". Это как аварийный тормоз.
- **Take-Profit** — "если цена выросла на Y%, автоматически продаём и фиксируем прибыль". Это как будильник: "хватит, забирай деньги".
- Уровни рассчитываются по ATR — чем волатильнее рынок, тем дальше стоп-лосс (чтобы не выбило случайным колебанием).

### Уровень 3: Лимит открытых позиций
Бот не может открыть больше **3 позиций одновременно**. Даже если все стратегии кричат "покупай!" — после 3 сделок бот говорит "стоп, хватит".

### Уровень 4: Дневной лимит убытков
Если за день бот потерял больше **5% от баланса** — он полностью останавливает торговлю до следующего дня. Это как лимит в казино: проиграл дневной бюджет — иди домой.

### Уровень 5: Режимы состояния (светофор)
Бот работает в 4 режимах, как светофор:
- 🟢 **NORMAL** — всё хорошо, торгуем в полную силу
- 🟡 **REDUCED** — что-то не так (например, много убытков подряд), размер позиций уменьшается вдвое
- 🟠 **SAFE** — серьёзные проблемы, только закрываем существующие позиции, новые не открываем
- 🔴 **STOP** — критическая ситуация, всё останавливается. Нужно вмешательство человека.

Переходы между режимами автоматические: 3 убытка подряд → REDUCED, 5 убытков → SAFE, дневной лимит превышен → STOP.

### Уровень 6: Автоматические выключатели (Circuit Breakers)
Это 8 независимых "предохранителей", каждый из которых может заблокировать торговлю:
1. Слишком быстрое падение цены (flash crash)
2. Слишком низкая ликвидность (объёмы торгов упали)
3. Слишком высокая волатильность
4. Много убыточных сделок подряд
5. API биржи работает медленно
6. Потеря связи
7. Спред (разница между покупкой и продажей) слишком большой
8. Аномальный объём (возможна манипуляция)

Каждый предохранитель включается и выключается независимо, как автоматы в электрощитке.

### Уровень 7: Kill Switch (аварийная кнопка)
Если всё совсем плохо — есть кнопка, которая мгновенно закрывает ВСЕ позиции и останавливает бота. Это можно сделать через Telegram или через веб-панель.

---

## Как бот учится на своих ошибках?

После каждой закрытой сделки бот записывает результат в базу данных. ML-предиктор периодически переобучается на новых данных. То есть:
- Бот сделал 100 сделок за неделю
- Записал все параметры (какие были индикаторы, какой был результат)
- ML-модель обучается на 60% данных, проверяется на 20%, тестируется на 20%
- Если модель хорошая (точность > 55%) — она начинает фильтровать сигналы
- Если плохая — бот торгует без ML-фильтра (только по стратегиям)

---

## Как бот отчитывается?

1. **Telegram-бот** — отправляет уведомления о каждой сделке, ежедневную статистику, предупреждения об ошибках. Можно отправить команду и узнать текущий статус.

2. **Веб-панель (Dashboard)** — открывается в браузере на порту 8888. Показывает:
   - Открытые позиции
   - Историю сделок
   - Текущие новости и их влияние
   - Общую прибыль/убыток
   - Статус всех систем

3. **Логи** — всё записывается в файлы для анализа.

---

## Важно понимать

1. **Бот не гарантирует прибыль.** Никакой робот не может предсказать рынок на 100%. Sentinel спроектирован так, чтобы **минимизировать убытки** и **максимизировать шансы** на прибыль, но риск всегда есть.

2. **Бот работает в "бумажном" (paper) режиме.** Это значит, что он не торгует настоящими деньгами — он симулирует сделки. Это для тестирования и обучения.

3. **Чем больше данных — тем лучше.** ML-предиктор становится точнее с каждой сделкой. Первые несколько дней бот может ошибаться чаще.

4. **Все пороги настраиваемые.** Всё, что написано выше (порог уверенности 0.6, лимит 3 позиции, 5% дневной лимит) — это настройки, которые можно менять в конфигурации.

## 12. ОГРАНИЧЕНИЯ И EDGE CASES

1. **Cold start**: Нет торговли пока не набрано 55 свечей (≈55 часов 1h данных)
2. **ML cold start**: ML не активен до 500 сделок
3. **Stale data**: Features обновляются только при закрытии 1h свечи
4. **4h lag**: 4h данные обновляются каждые 4 часа
5. **Regime lock**: Режим меняется только на новой свече, не на тике
6. **Paper slippage**: Исполнение по market price без учёта реального slippage
7. **Commission**: Фиксированная 0.1% per side (paper mode)
8. **Single signal**: Только 1 сигнал на символ за свечу (break после первого)
9. **No pyramiding**: Максимум 2 позиции одновременно
10. **News latency**: RSS обновляется каждые 5 минут
11. **NaN/Inf**: Все значения санитизируются через safe_value() → 0.0
12. **Grid breakage**: Если цена вне сетки ±2%, торговля сетки замирает

## 13. КРИТИЧЕСКИЕ НАБЛЮДЕНИЯ

1. **Volume — главный подтверждающий фактор**: каждая стратегия использует volume_ratio, даёт +0.05-0.10 confidence
2. **Тройной фильтр тренда**: EMA50(4h) + ADX > 25 + EMA9 > EMA21 = сильнейшее подтверждение
3. **Ассиметрия SL/TP confidence**: BUY 0.75-0.95, SELL 0.75-0.99 — sell-сигналы всегда более уверенные
4. **Mean Reversion — единственная контрарная**: страх увеличивает confidence (остальные — наоборот)
5. **DCA — детерминистическая**: не зависит от confidence, только от времени и dip-множителя
6. **ML — только вето**: не генерирует сигналы, только блокирует плохие
7. **Sentiment — умеренное влияние**: ±0.10 confidence = ~13% от рабочего диапазона (0.75-0.95)
8. **Position sizing — тройной фильтр**: Kelly × ATR volatility × ADX dampener
9. **3 уровня risk**: 7 checks + 4-state machine + 8 circuit breakers
10. **Прогресивная деградация**: NORMAL → REDUCED → SAFE → STOP при росте потерь
