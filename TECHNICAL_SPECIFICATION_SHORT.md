# AI QUANT TRADING SYSTEM — "SENTINEL"

## 1. Цель проекта

Построить домашнюю AI-систему для торговли на Binance Spot, которая:

- работает только на споте без плеча и фьючерсов;
- сначала торгует в paper-режиме;
- использует строгий риск-контроль;
- обучается на истории собственных сделок;
- может быть постепенно переведена в live-режим с малым капиталом.

Целевой ориентир версии V1.5: выйти на дополнительный доход около **$200/мес** в горизонте **15–18 месяцев** при внешнем пополнении капитала и без ослабления risk-limits.

---

## 2. Профиль и ограничения

**Пользователь:** начинающий трейдер и начинающий Python-разработчик.  
**Стартовый капитал:** $100–500.  
**Абсолютный риск-лимит:** до $50 дневного убытка как операционная граница.  
**Биржа:** Binance.  
**Инструменты:** только BTC/USDT и ETH/USDT.  
**ОС:** Windows.  
**Рынок:** только Spot.

Жёсткие ограничения:

- без фьючерсов;
- без маржи;
- без leverage;
- без права вывода средств через API;
- только buy/sell на споте.

---

## 3. Основные принципы

Приоритеты системы:

1. Безопасность.
2. Контроль со стороны пользователя.
3. Стабильность работы.
4. Предсказуемость логики.
5. Прибыль.

Ключевое правило:

**Система не должна сама переводить себя на следующий этап торговли.**  
Переход между режимами делает только пользователь после анализа результатов.

Важно:

- система ограничивает открытие нового риска, но не может математически гарантировать отсутствие убытка при гэпах, проскальзывании, сбоях API и проблемах связи;
- live-режим допустим только при наличии биржевых защитных ордеров или эквивалентной защиты на стороне биржи.

---

## 4. Режимы работы

1. **Signal Mode**  
   Анализ рынка и отправка сигналов без автоматического исполнения.

2. **Paper Trading**  
   Основной стартовый режим: реальные рыночные данные, но виртуальные сделки.

3. **Live Micro**  
   Реальная торговля на малом капитале ($50–100) с жёсткими лимитами.

4. **Live Full**  
   Полноценный автоматический режим на всём рабочем капитале.

## 5. Архитектура системы

Базовый поток данных:

**Binance API → Data Collector → SQLite → Feature Engine → Strategy Engine → Risk Sentinel → Execution Engine → Position Manager / Dashboard / Telegram**

Технологический стек:

## 6. Ключевые модули

### Data Collector
Собирает сделки, свечи и вспомогательные рыночные данные с Binance в реальном времени.

### Database Layer
Хранит trades, candles, signals, positions, результаты paper/live торговли и статистику.

### Feature Engine
Считает признаки и индикаторы: EMA, RSI, MACD, объёмы, режим рынка и другие производные признаки.

### Strategy Engine
Формирует торговые сигналы на базе набора стратегий.

В версии V1.5 предусмотрен набор из 6 стратегий:

- EMA / RSI базовая;
- Grid Trading;
- Mean Reversion;
- Bollinger Breakout;
- DCA Bot;
- MACD Divergence.

### Risk Sentinel
Проверяет каждый вход по risk-лимитам, частоте сделок, состоянию системы и текущему режиму.

### Execution Engine
Исполняет сделки в paper или live режиме, учитывая комиссии, проскальзывание и статус ордеров.

### Position Manager
Следит за открытыми позициями, PnL, SL/TP, историей сделок и закрытием позиций.

### Backtest Engine
Проверяет стратегии на исторических данных с учётом комиссий и проскальзывания.

### Telegram Bot
Даёт сигналы, статус, PnL, алерты и команды управления.

### Web Dashboard
Показывает состояние системы, сделки, позиции, график PnL и базовое управление.

### Trade Analyzer
Модуль самообучения V1.5. Анализирует закрытые сделки, ищет ошибки, оценивает качество сигналов и помогает корректировать стратегии по истории собственных результатов.

## 7. Управление риском

Система должна включать:

- дневной лимит убытка;
- ограничение количества сделок за период;
- kill-switch;
- state machine режимов `NORMAL -> REDUCED -> STOP`;
- circuit breakers;
- watchdog на отказ модулей;
- проверку целостности данных;
- защиту от некорректных или повреждённых входных данных.

Подход к риску:

- сначала остановить новые входы;
- затем уведомить пользователя;
- затем перевести систему в безопасное состояние.

## 9. Минимальные критерии успеха

Систему можно считать готовой к следующему этапу, если:

- paper-режим стабильно работает без критических сбоев;
- risk-лимиты реально останавливают новые входы;
- все сделки, сигналы и ошибки логируются;
- результаты backtest и paper согласуются по логике;
- пользователь видит статус системы и может остановить её вручную;
- Trade Analyzer даёт понятную аналитику по ошибкам и качеству сделок.

## 10. Итог

`Sentinel` — это не просто торговый бот, а контролируемая полуавтоматическая инфраструктура для безопасного входа новичка в алгоритмический спот-трейдинг.  
Его главная идея: **сначала дисциплина и контроль риска, потом масштабирование и прибыль.**
