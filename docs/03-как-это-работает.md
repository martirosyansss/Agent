# Как это работает

Общий цикл — сверху вниз, без формул.

## Что происходит каждый час

Бот работает по закрытию каждой часовой свечи на Binance. В этот
момент выполняется цепочка:

```
новые цены          →  посчитать индикаторы
                       (RSI, MACD, ADX, Bollinger, ATR, …)
                          ↓
определить режим    →  тренд / боковик / волатильность
                          ↓
спросить стратегии  →  EMA-crossover? Bollinger? MACD-divergence?
                       Mean-reversion? Grid? DCA?
                          ↓
если есть сигнал    →  ML-фильтр: «это похоже на выигрышную
                         сделку по прошлому опыту?»
                          ↓
если ML разрешил    →  риск-проверки: сколько уже открыто,
                         сколько потеряли сегодня, нет ли
                         аномалий на рынке
                          ↓
если риск разрешил  →  открыть позицию (paper или live)
                          ↓
следить за позицией →  стоп-лосс, тейк-профит, trailing,
                         вовремя закрыть
                          ↓
записать результат  →  в базу SQLite для последующего обучения ML
```

## Стратегии

В боте несколько независимых «мнений» — каждая стратегия смотрит
на цену со своей стороны и говорит «вот тут вход, вот причина».

| Стратегия | Идея |
|---|---|
| **EMA crossover + RSI** | Короткая скользящая пересекает длинную + подтверждение RSI. Классический «кроссовер». |
| **Bollinger breakout** | Цена пробивает верхнюю / нижнюю ленту на высоком объёме — ловим импульс. |
| **MACD divergence** | Классическая дивергенция между ценой и MACD — сигнал смены тренда. |
| **Mean reversion** | RSI в перепроданности в боковике → покупка. Работает в range-режиме, невыгодна в тренде. |
| **Grid trading** | Расставить сетку ордеров в коридоре цены. Зарабатывает на мелких колебаниях, если рынок боковой. |
| **DCA bot** | Постепенное усреднение на падающей цене. Долгосрочная накопительная логика. |

Каждая из стратегий **может быть включена или выключена в `.env`**:

```
GRID_ENABLED=true
MEANREV_ENABLED=false
```

По умолчанию включены только EMA-crossover, Bollinger и MACD — три
классики, которые лучше всех работают на часовом таймфрейме в крипте.

## Режимы рынка (regime)

Прежде чем слушать стратегии, бот определяет текущий **режим рынка**
по ADX (индикатор тренда) и Bollinger-ширине:

- **trending_up** — устойчивый рост, ADX > 25, EMA9 > EMA21
- **trending_down** — устойчивое падение
- **sideways** — боковик, ADX < 20, узкий Bollinger
- **volatile** — высокий ATR, широкие ленты, резкие движения
- **transitioning** — переходное состояние, ADX между 20 и 25

**Это важно**: одна и та же стратегия ведёт себя по-разному в
разных режимах. Mean-reversion, например, отлично работает в
боковике и катастрофически в тренде. Бот это знает — в трендовом
режиме он даёт меньший вес mean-reversion-сигналам.

## Multi-timeframe gate

Сигнал на часовом графике проверяется на **4-часовом**: если на
1h бот хочет купить, но на 4h всё падает — сигнал отклоняется.
Это снижает вход «против тренда старшего таймфрейма».

## ML-фильтр

ML-модель **не генерирует сигналы**. Она только фильтрует то, что
сгенерировали стратегии: «этот сигнал похож на выигрышный или на
проигрышный, судя по прошлым сделкам?».

Результат модели — вероятность от 0 до 1. Если:

- **> 0.60** → сделка разрешена
- **0.47–0.60** → «reduce» (уменьшить размер в 2 раза)
- **< 0.47** → блокировка

Перед каждым предсказанием бот проверяет **PSI-дрейф** — не
изменились ли живые распределения фичей по сравнению с
обучающими. Если сильно ушли — запланирует внеплановое
переобучение.

Подробнее — в [отдельной странице про ML](05-машинное-обучение.md).

## Риск-контроль

Даже если стратегия и ML оба сказали «да», ещё не факт, что
сделка откроется. Дальше сигнал проходит через серию **гейтов**:

1. **Лимит дневных потерь**: если сегодня уже потеряли $50 —
   новые сделки отклоняются до следующего дня.
2. **Лимит открытых позиций**: не больше 2 одновременно.
3. **Лимит на размер**: в одну позицию — не больше 20% депозита.
4. **Лимит по корреляции**: нельзя одновременно держать BTC и ETH
   в одну сторону (они коррелированы, это удвоенный риск).
5. **Circuit breakers**: если на рынке аномалия (цена прыгнула на
   5% за минуту, объём в 10 раз выше обычного, API Binance стал
   отвечать с задержкой > 5 сек) — бот замораживает торговлю
   до возврата в норму.

Если хоть один гейт сказал «нет» — сигнал отклоняется. Подробности
про каждый — в [документе про безопасность](04-безопасность.md).

## Исполнение и сопровождение

Когда сделка всё-таки открывается, включается **пятиступенчатый
Professional Exit** (все фазы работают одновременно):

1. **Базовый SL/TP** — фиксированные стоп и тейк на бирже.
2. **Динамический SL** — каждую свечу бот пересматривает стопы
   с учётом ATR и текущего режима.
3. **Жёсткий выход по регим-флипу** — если HMM подтверждённо
   сказал «тренд развернулся» — закрываем не дожидаясь стопа.
4. **Chandelier Exit** — ратчет-трейлинг: стоп подтягивается за
   максимумом цены, не опускается назад.
5. **Weekend Exit Guard** (опционально) — пятница 20:00 UTC →
   понедельник 00:00 UTC: всё закрыть, новых не открывать.

Плюс **scale-out ladder**: частичные закрытия на промежуточных
уровнях (+1R, +2R, +3R) — гарантированная прибыль плюс шанс
поймать крупное движение остатком. Ладдер у каждой стратегии свой.

Подробнее — в [04-безопасность](04-безопасность.md).

## Что записывается

Каждая сделка, каждое решение ML-модели, каждое срабатывание
риск-гейта — всё пишется в SQLite-базу `sentinel/data/sentinel.db`.
Это нужно для двух вещей:

1. **Аудита**: вы можете открыть dashboard и посмотреть «почему
   вчера в 14:00 бот отказался покупать».
2. **Дообучения ML**: каждые 2 недели модель автоматически
   переучивается на новых сделках, становясь всё более
   настроенной под текущий рынок.
