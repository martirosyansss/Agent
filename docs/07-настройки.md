# Настройки

Все настройки бота живут в **одном файле** — `sentinel/.env`. После
любого изменения нужно **перезапустить бота** (Ctrl+C → `python main.py`),
без этого новые значения не подхватятся.

Ниже — все параметры, сгруппированные по назначению. Значения по
умолчанию подобраны для консервативного paper-режима с депозитом
$100–500.

## Binance

```
BINANCE_API_KEY=...
BINANCE_API_SECRET=...
```

Ключи с Binance. Для paper-режима — **только read-only**. Для live —
read + spot trading, **никогда не withdrawals**.

## Telegram (опционально)

```
TELEGRAM_BOT_TOKEN=...
TELEGRAM_CHAT_ID=...
TELEGRAM_PIN=1234
```

- Если `TOKEN` и `CHAT_ID` пустые — бот работает, просто не шлёт
  уведомления.
- `TELEGRAM_PIN` защищает опасные команды (`/pause`, `/close_all`).
  Если не задан — команды открыты любому, кто пишет боту.

## Торговля

```
TRADING_MODE=paper          # или live
TRADING_SYMBOLS=["BTCUSDT","ETHUSDT"]
```

- `paper` — без реальных денег. Начните с этого.
- `live` — реальные деньги. Только после 2+ недель в paper.
- `TRADING_SYMBOLS` — какие пары торгуем. По умолчанию только BTC
  и ETH. Добавлять больше пар не рекомендую, пока не разберётесь.

## Риск-лимиты

```
MAX_DAILY_LOSS_USD=50.0
MAX_DAILY_LOSS_PCT=10.0
MAX_POSITION_PCT=20.0
MAX_TOTAL_EXPOSURE_PCT=60.0
MAX_OPEN_POSITIONS=2
MAX_ORDER_USD=100.0
MAX_TRADES_PER_HOUR=2
MAX_TRADES_PER_DAY=6
```

Подробности — в [04-безопасность](04-безопасность.md). Главное
помните: **срабатывает то, что строже**. Если `MAX_DAILY_LOSS_USD=50`
и `MAX_DAILY_LOSS_PCT=10%` при депозите $300 — фактический лимит
$30 (10%), а не $50.

## Circuit breakers

```
CB_PRICE_ANOMALY_PCT=5.0
CB_CONSECUTIVE_LOSSES=3
CB_SPREAD_ANOMALY_PCT=0.5
CB_VOLUME_ANOMALY_MULT=10.0
CB_API_ERROR_COUNT=5
CB_LATENCY_THRESHOLD_SEC=5.0
CB_BALANCE_MISMATCH_PCT=1.0
CB_COMMISSION_ALERT_PCT=1.0
```

Пороги, после которых бот замораживает торговлю. Эти цифры менять
стоит только если точно понимаете, что делаете. Поднимая порог, вы
снижаете защиту.

## Стратегия

```
STOP_LOSS_PCT=3.0
TAKE_PROFIT_PCT=7.0
MIN_CONFIDENCE=0.75
SIGNAL_TIMEFRAME=1h
TREND_TIMEFRAME=4h
```

- `STOP_LOSS_PCT` — стоп-лосс 3% от цены входа.
- `TAKE_PROFIT_PCT` — тейк-профит 7%. Отношение TP/SL ≈ 2.33, это
  хорошо для swing-трейдинга.
- `MIN_CONFIDENCE` — минимальная уверенность стратегии (не ML) для
  входа. Ниже этого — сигнал отклоняется.
- `SIGNAL_TIMEFRAME` — таймфрейм для принятия решения (1 час).
- `TREND_TIMEFRAME` — таймфрейм для multi-timeframe gate (4 часа).

## Weekend Exit Guard (Phase 5 Professional Exit)

```
WEEKEND_EXIT_ENABLED=false
WEEKEND_EXIT_CUTOFF_DAY_OF_WEEK=4     # 0=Пн … 4=Пт, 6=Вс
WEEKEND_EXIT_CUTOFF_HOUR_UTC=20
WEEKEND_EXIT_REOPEN_DAY_OF_WEEK=0     # Понедельник
WEEKEND_EXIT_REOPEN_HOUR_UTC=0
```

Опционально: по пятницам в 20:00 UTC принудительно закрыть все
позиции и не открывать новых до понедельника 00:00 UTC. Защита от
тонкой ликвидности и flash-crash-ей в выходные. По умолчанию
**выключено**, чтобы не менять поведение после апгрейда.

Подробнее — в [04-безопасность](04-безопасность.md) раздел
«Professional Exit».

## Включение стратегий

```
GRID_ENABLED=false
MEANREV_ENABLED=false
BB_BREAKOUT_ENABLED=false
DCA_ENABLED=false
MACD_DIV_ENABLED=false
```

По умолчанию включены только **EMA crossover + RSI** (встроенная,
без отдельного флага). Остальные стратегии отключены — их включают
по одной, **проверяя результат минимум 2 недели**. Включать всё
сразу — плохая идея: запутаетесь, что дало плюс, а что минус.

### Grid (торговля сеткой)

```
GRID_NUM_LEVELS=8
GRID_CAPITAL_PCT=30.0
GRID_AUTO_RANGE=true
GRID_MIN_PROFIT_PCT=0.3
GRID_MAX_LOSS_PCT=5.0
```

Работает только в боковике. Если рынок трендовый — стратегия
простаивает (это нормально).

### Mean Reversion

```
MEANREV_RSI_OVERSOLD=25.0
MEANREV_RSI_OVERBOUGHT=75.0
MEANREV_STOP_LOSS_PCT=4.0
MEANREV_TAKE_PROFIT_PCT=6.0
MEANREV_CAPITAL_PCT=15.0
```

Покупаем на RSI<25 в боковике, продаём на RSI>75. **Катастрофически
не работает в тренде** — поэтому имеет смысл включать вместе с
`AUTO_STRATEGY_SELECTION=true`, чтобы она автоматически выключалась
в трендовом режиме.

### Bollinger Breakout

```
BB_PERIOD=20
BB_STD_DEV=2.0
BB_VOLUME_CONFIRM_MULT=1.5
BB_SQUEEZE_THRESHOLD=0.05
BB_STOP_LOSS_PCT=3.0
BB_TAKE_PROFIT_PCT=6.0
BB_TRAILING_STOP_PCT=2.0
```

Классический пробой Bollinger-лент с подтверждением объёмом. Хорошо
работает на волатильных движениях.

### DCA (dollar cost averaging)

```
DCA_BASE_AMOUNT_USD=10.0
DCA_INTERVAL_HOURS=24
DCA_MAX_DAILY_BUYS=3
DCA_MAX_INVESTED_PCT=40.0
DCA_STOP_DRAWDOWN_PCT=15.0
DCA_TAKE_PROFIT_PCT=8.0
DCA_PARTIAL_TP_PCT=5.0
```

Долгосрочная стратегия — регулярные покупки на падении. Не для
быстрой прибыли.

### MACD Divergence

```
MACD_FAST=12
MACD_SLOW=26
MACD_SIGNAL_PERIOD=9
MACD_LOOKBACK_CANDLES=30
MACD_DIV_STOP_LOSS_PCT=3.5
MACD_DIV_TAKE_PROFIT_PCT=7.0
MACD_REQUIRE_RSI_CONFIRM=true
MACD_REQUIRE_VOL_CONFIRM=true
```

Ищет расхождения между ценой и MACD — классический сигнал смены
тренда.

### Auto strategy selection

```
AUTO_STRATEGY_SELECTION=false
REGIME_CHECK_INTERVAL_HOURS=4
ADX_TRENDING_THRESHOLD=25.0
ADX_SIDEWAYS_THRESHOLD=20.0
```

Если `true`, бот **сам** включает/выключает стратегии в зависимости
от текущего режима рынка: mean-reversion — только в боковике,
trend-following — только в тренде.

## Machine Learning

```
ANALYZER_ML_ENABLED=true
ANALYZER_ML_SHADOW_MODE=true
ANALYZER_MIN_TRADES_ML=200
ANALYZER_ML_RETRAIN_DAYS=14
ANALYZER_ML_BLOCK_THRESHOLD=0.60
ANALYZER_ML_HISTORY_DAYS=180
ANALYZER_ML_TEST_WINDOW_DAYS=60
ANALYZER_ML_MIN_SKILL_SCORE=0.72
ANALYZER_ML_MIN_PRECISION=0.65
ANALYZER_ML_MIN_RECALL=0.58
ANALYZER_ML_MIN_ROC_AUC=0.65
ANALYZER_ML_USE_WALK_FORWARD=true
ANALYZER_ML_USE_BOOTSTRAP_CI=true
ANALYZER_ML_USE_STACKING=false
ANALYZER_ML_USE_REGIME_ROUTING=false
```

- `ANALYZER_ML_ENABLED=true` — ML-фильтр включён. Если сделок
  меньше порога, фактически работать всё равно не начнёт.
- `ANALYZER_ML_SHADOW_MODE=true` — ML считает, но не влияет на
  реальные решения. Рекомендую держать так **первую неделю** после
  накопления данных, чтобы посмотреть, какие сигналы он хочет
  заблокировать.
- `ANALYZER_MIN_TRADES_ML=200` — меньше 200 сделок в истории —
  ML не учим.
- `ANALYZER_ML_RETRAIN_DAYS=14` — плановое переобучение раз в 2
  недели (или раньше, если PSI-мониторинг поймал дрейф).
- `ANALYZER_ML_BLOCK_THRESHOLD=0.60` — порог вероятности, ниже
  которого сигнал блокируется (с «reduce»-зоной ниже).
- `ANALYZER_ML_MIN_ROC_AUC=0.65`, `..._MIN_PRECISION=0.65`,
  `..._MIN_RECALL=0.58`, `..._MIN_SKILL_SCORE=0.72` — минимальные
  метрики; не дотягивает — модель отвергается, остаётся старая.
- `ANALYZER_ML_USE_WALK_FORWARD=true` — walk-forward CV с
  purge/embargo (стандарт López de Prado).
- `ANALYZER_ML_USE_BOOTSTRAP_CI=true` — доверительные интервалы на
  метриках через bootstrap.
- `ANALYZER_ML_USE_STACKING=false` — опционально: stacking-голова
  над ансамблем. Требует включённого walk-forward.
- `ANALYZER_ML_USE_REGIME_ROUTING=false` — опционально: отдельные
  модели на каждый режим. Требует ≥100 сделок на режим.

Отдельно действует **PSR/DSR gate** — модель не уходит в
production, если PSR < 0.95 или DSR < 0.95. Порог фиксированный в
коде, не настраивается через `.env`.

Подробнее — в [05-машинное-обучение](05-машинное-обучение.md).

## Paper trading

```
PAPER_INITIAL_BALANCE=500.0
PAPER_COMMISSION_PCT=0.1
PAPER_SLIPPAGE_PCT=0.05
```

Параметры имитации: начальный баланс, комиссия за сделку (0.1% —
как у Binance спот), slippage (проскальзывание — реальная цена
исполнения часто на 0.05% хуже той, что видна). Всё это считается
при paper-сделке, чтобы результаты были **реалистичные**, а не
идеализированные.

## Система

```
LOG_LEVEL=INFO
DASHBOARD_PORT=8080
DASHBOARD_PASSWORD=
DB_PATH=data/sentinel.db
DB_BACKUP_INTERVAL_HOURS=6
MAX_RAM_MB=2048
```

- `LOG_LEVEL`: `DEBUG` / `INFO` / `WARN` / `ERROR`. Для отладки ставим
  `DEBUG`, в обычной работе — `INFO`.
- `DASHBOARD_PORT`: если 8080 занят — поменяйте.
- `DASHBOARD_PASSWORD`: если бот не на `127.0.0.1`, обязательно.
- `DB_BACKUP_INTERVAL_HOURS=6`: каждые 6 часов бот копирует базу в
  `sentinel.db.backup_<timestamp>`. Полезно при сбоях.
- `MAX_RAM_MB=2048`: если процесс съест больше 2 ГБ, watchdog
  аварийно перезапустит его.

## Внешний мониторинг (опционально)

```
SENTINEL_LOG_SHIP_URL=
```

Если задано — бот батчами шлёт `events.jsonl` на внешний webhook
(S3-совместимый bucket, собственный сборщик логов). Пусто — никуда
не шлёт. Полезно, если бот крутится на VPS и вы не хотите терять
журналы при откате диска.

Алертинг в Telegram настраивается уже существующими
`TELEGRAM_BOT_TOKEN` / `TELEGRAM_CHAT_ID`. Если они заданы, бот
автоматически шлёт push на критические события (CB-срабатывания,
кил-свитчи, аномалии), с cooldown-ом от спама.

## Cooling / первый день live

```
RESUME_COOLDOWN_MIN=30
LIVE_FIRST_DAY_MAX_ORDER=20.0
```

- После срабатывания circuit breaker — 30 минут тишины перед
  возвратом в работу.
- В первые 24 часа live-режима размер ордера ограничен $20 — защита
  от забытых риск-лимитов.

## Что изменять осторожно

Это — список параметров, где **увеличение значения = повышение риска**:

- `MAX_DAILY_LOSS_USD`, `MAX_DAILY_LOSS_PCT` (больше можно потерять)
- `MAX_POSITION_PCT`, `MAX_ORDER_USD` (больше одна сделка)
- `MAX_OPEN_POSITIONS` (больше одновременных позиций)
- `MAX_TRADES_PER_HOUR`, `MAX_TRADES_PER_DAY` (больше попыток ловить шум)
- `CB_*` (поднимая — снижаете защиту)
- `STOP_LOSS_PCT` (больший стоп = большая возможная потеря)

Это — **не** причина их не менять никогда. Это — причина менять
**постепенно** и с пониманием, что делаете.
