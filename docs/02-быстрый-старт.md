# Быстрый старт

Цель этой страницы — за 15 минут запустить бота в безопасном
paper-режиме. Реальные деньги не понадобятся.

## Что потребуется

- Компьютер с Windows, Linux или macOS, который может 24/7 быть
  включён (иначе бот ничего не поторгует).
- Python 3.12 или новее.
- Аккаунт на Binance (пусть даже пустой — для paper-режима это
  нормально).
- Аккаунт в Telegram (для уведомлений — опционально, но очень
  рекомендую).

## Шаг 1: скачать код

```powershell
git clone <ваш-репозиторий>
cd Agent
```

Все команды в этой инструкции — из папки `c:/Softs/Agent` (или где
у вас лежит проект).

## Шаг 2: поставить Python-зависимости

```powershell
cd sentinel
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
```

На macOS / Linux третья строка выглядит так: `source venv/bin/activate`.

Установка может занять 3–5 минут — ставится numpy, scikit-learn,
lightgbm, xgboost и ещё пара десятков пакетов.

## Шаг 3: создать `.env`

В папке `sentinel/` уже лежит файл `.env.example` — скопируйте его
в `.env`:

```powershell
copy .env.example .env
```

Затем откройте `.env` в любом текстовом редакторе.

**Для paper-режима** можно оставить все значения по умолчанию и
только добавить Binance API-ключи (даже с read-only правами):

```
BINANCE_API_KEY=ваш_ключ
BINANCE_API_SECRET=ваш_секрет
TRADING_MODE=paper
```

Как получить API-ключи на Binance:

1. Войти на binance.com → Profile → API Management.
2. Create API → дать имя → включить только **Read Info**
   (никаких Trade / Futures, нам они сейчас не нужны).
3. Скопировать `API Key` и `Secret Key` в `.env`.

**Почему только read**: в paper-режиме бот берёт с Binance только
цены, он не ставит реальные ордера. Read-only ключ — это страховка:
даже если `.env` случайно утечёт, никто ничего не купит от вашего
имени.

## Шаг 4: (опционально) Telegram-уведомления

Без этого бот тоже работает, но удобнее видеть сделки и ошибки
прямо в телефоне.

1. Найти в Telegram бота **@BotFather** → `/newbot` → придумать
   имя → получить `TOKEN`.
2. Найти своего нового бота в Telegram → написать ему что-нибудь.
3. Открыть `https://api.telegram.org/bot<ВАШ_TOKEN>/getUpdates` в
   браузере — там будет ваш `chat_id` (число).
4. В `.env`:

```
TELEGRAM_BOT_TOKEN=токен_от_BotFather
TELEGRAM_CHAT_ID=ваш_chat_id
```

## Шаг 5: запустить

```powershell
python main.py
```

В консоли побегут логи. Если всё хорошо — через 10–20 секунд
увидите что-то вроде:

```
[Module] ML Predictors: 2 per-symbol + unified fallback (mode=shadow)
[Module] Dashboard available at http://127.0.0.1:8080
[Module] Preflight OK — entering trading loop
```

## Шаг 6: открыть dashboard

В браузере: **http://127.0.0.1:8080**

Там вы увидите текущие позиции (пока пусто), последние сигналы,
состояние ML-модели, и логи в реальном времени.

Подробно про экраны dashboard — в [отдельной странице](06-dashboard.md).

## Что дальше

Оставьте бота работать **минимум 2 недели в paper-режиме**. За это
время он:

- соберёт историю сделок (в `sentinel/data/sentinel.db`),
- проверит свою ML-модель на реальных ценах,
- покажет, какая у него статистика win-rate и средней прибыли.

Только после этого можно задумываться о переходе в live —
см. [документ про безопасность](04-безопасность.md).

## Если что-то не запустилось

Сначала проверьте:

- Python 3.12+: `python --version`
- Все зависимости стали: `pip list | grep numpy` — должен быть numpy.
- В `.env` ключи Binance заполнены и без лишних пробелов / кавычек.
- Порт 8080 свободен (или поменяйте `DASHBOARD_PORT` в `.env`).

Если всё ещё не работает — [страница troubleshooting](09-если-что-то-пошло-не-так.md).
